🎯 **What:** Removed the assignment of the `Future` returned by the background metadata fetch to an unused variable (`metadataLookup`) in `lib/screens/watch_screen.dart`.

💡 **Why:** To resolve an "Unused Declaration" code health issue. Storing a Future in a local variable without ever awaiting it or using it adds clutter and triggers linting warnings.

✅ **Verification:**
- Evaluated the impact: The `Future` is meant to be a fire-and-forget side effect (updating state in parallel). Removing the variable assignment safely preserves this behavior exactly.
- Ran `dart analyze` to verify the "Unused Declaration" warning is resolved.
- Ran `flutter test` to ensure existing functionality is unaffected.

✨ **Result:** Improved code cleanliness and readability by removing dead code while preserving correct concurrent execution.
